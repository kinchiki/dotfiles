# run-verification

verification ドキュメントを 1 項目ずつ進め、結果をその場でファイルへ反映するスキルです。
完了済み項目は飛ばし、未完了の `□` だけを再開地点として扱います。

## Scope

- 現在のブランチまたは指定パスに対応する verification ファイルを特定する。
- `## 確認N` の内容を順に案内する。
- ユーザー入力に応じて `## チェックリスト` の `□` を `✅` または `⏭️` に更新する。
- 中断後に再開できる状態を保つ。

## Hard constraints

- 引数なしの場合は、現在のブランチ名から `.ai-loacl/verification/` 内の候補を探す。
- 候補が複数ある場合は、勝手に選ばずファイル名を提示してユーザーに選んでもらう。
- 対応ファイルが見つからない場合は、新規作成せず `create-verification` を案内して止まる。
- 既に `✅` または `⏭️` の項目は再実行対象にしない。
- `ok` 系の返答だけを `✅`、`skip` 系の返答だけを `⏭️` にする。
- 質問や相談が返ってきた場合は回答した上で、同じ項目に留まる。

## Workflow

### Step 1: Resolve the verification file

- 現在のブランチ名を取得する。
- 引数がファイルパスならそのファイルを使う。
- 引数が無い場合は、ブランチ名をスラッグ化して `.ai-loacl/verification/` の候補を探す。

### Step 2: Read the current state

- verification ファイルを読み、前提、確認セクション、チェックリストの対応を把握する。
- `✅` と `⏭️` を済みとして扱い、残りの `□` を抽出する。

### Step 3: Guide each remaining check

- 未完了の確認を 1 件ずつそのまま提示する。
- `ok` / `OK` / `はい` / `完了` なら該当チェックを `✅` にする。
- `skip` / `スキップ` なら該当チェックを `⏭️` にする。
- それ以外の返答には回答し、同じ確認を続ける。

### Step 4: Summarize the current state

- すべて案内し終えたら、`✅`、`⏭️`、`□` の件数を集計して報告する。
- `□` が残る場合は、同じファイルを指定して再開できることを案内する。

## Report

日本語で次を報告してください。

- 対象 verification ファイルの path
- 今回 `✅` にした件数
- 今回 `⏭️` にした件数
- 残っている `□` の件数
